## ⭐ Hệ Thống Đánh Giá

Mỗi truyện được chấm điểm theo 4 tiêu chí (thang 10):

| Tiêu chí | Mô tả |
|----------|--------|
| 🎭 Tính bất ngờ | Twist ending có mạnh không? Punchline có khó đoán? |
| 🇻🇳 Phù hợp văn hóa Việt | Người Việt có đồng cảm không? Context có gần gũi? |
| 💡 Dễ hiểu | Không cần giải thích thêm? Hiểu ngay lần đầu? |
| 🗣️ Khả năng kể lại | Có thể kể cho bạn bè nghe? Dễ nhớ, dễ truyền? |

**Ngưỡng chấp nhận:**
- **≥ 30/40** → Đưa vào sách ✅
- **≥ 35/40** → Truyện hay đặc biệt ⭐
- **< 30/40** → Loại ❌ (lưu vào [`rejected/rejected-log.md`](rejected/rejected-log.md))

Chi tiết chấm điểm từng truyện: [`metadata/scoring-log.md`](metadata/scoring-log.md)

## 🛠 Cách Build Sách

**Phụ thuộc:** [pandoc](https://pandoc.org/), TeX Live (xelatex cho PDF), font **Noto Sans** (tiếng Việt).

```bash
# Ghép tất cả chương thành full-book.md (có ngắt trang giữa các chương cho PDF)
bash scripts/build-book.sh

# Export sang PDF / EPUB / DOCX
bash scripts/export.sh
```

(`kich-ban/build-book.sh` và `kich-ban/export.sh` vẫn gọi các script trên để tương thích cũ.)

### Release trên GitHub

Vào **Actions** → workflow **Release sách (PDF / EPUB / DOCX)** → **Run workflow**: nhập **tag** (vd: `v1.0.0`), tùy chọn tiêu đề / draft / prerelease. Workflow sẽ build và đính kèm các file có tên theo tag, ví dụ `Cuoi-Vo-Bung-v1.0.0.pdf` (và `.epub`, `.docx`) vào trang **Releases** của repo.

## 📝 Đóng Góp

Dự án hoan nghênh đóng góp! Nếu bạn có truyện cười hay:

1. Fork repo này
2. Thêm truyện vào chương phù hợp theo [format chuẩn](CLAUDE.md#4-format-truyện-trong-sách)
3. Chấm điểm truyện theo [hệ thống đánh giá](#-hệ-thống-đánh-giá) (cần ≥ 30/40)
4. Tạo Pull Request

**Lưu ý quan trọng:**
- Truyện phải được Việt hóa theo [quy tắc Việt hóa](CLAUDE.md#3-quy-tắc-việt-hóa-localization-guidelines)
- Không chấp nhận nội dung 18+, phân biệt chủng tộc, nhạy cảm chính trị
- Mỗi truyện cần kèm bình luận hài hước `> 😂💬 *...*`
